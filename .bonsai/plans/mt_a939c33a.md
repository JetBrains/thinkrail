## Planning Decisions

Captured from the planning conversation. The description spec (`feature_login_screen.md`) was updated to match these decisions, so the plan and the spec are consistent.

| Decision | Choice | Implication |
|----------|--------|-------------|
| `/api/projects/known` | Keep tokenless REST | `ProjectPicker` continues to fetch recents over REST; no RPC migration. |
| v2→v3 migration | DROP+RENAME in place | `bonsai.db` survives; users/tokens/preferences/recent-projects tables dropped; `server_config` renamed to `settings`; `projects` table preserved. |
| Deprecated specs | **Delete entirely** (no audit-trail retention) | `module-auth-migration`, `feature-admin-system`, `module-user-api`, `feature_mobile_login`, `feature_preferences_sync` are removed via `spec_delete`. Description spec updated to match — git history is the audit trail. |
| Plan depth | Full detail | Each step is sized for one agent session. |

## Milestone 1: Backend — Storage Rename & Auth Removal

Rip out the auth, admin, and per-user storage layer from the Python backend. This is the deepest-touching milestone — every reference to `ServerStore` / tokens / `user_id` must move. Ordered so that the storage rename lands first; subsequent steps update callers.
