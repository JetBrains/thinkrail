Phase-Adaptive Explorations (v6)

Three interpretations of "where do artifacts live relative to the conversation?"

F1: Living Header
Artifacts Above, Conversation Below

Two-zone layout: a "living header" at the top evolves per phase (collapsed summary for Describe, diff viewer for Specify, plan steps for Plan). The conversation always fills the rest below. Header can be collapsed for pure chat feel. Directly implements: "artifacts on top of chat or as heading of chat".

KEY: Header grows/shrinks organically. Collapse it → pure Claude Code chat. Expand it → phase-specific artifact workspace above the conversation.
F2: Inline Journal
One Scrolling Stream

Everything in a single vertical timeline — like Claude Code or a notebook. Artifacts appear as rich blocks in the conversation flow. Phase transitions are subtle dividers. A floating TOC minimap appears in the right margin for navigation. Closest to "one chat in Claude Code" for simple tickets.

KEY: Simplest mental model. One scroll, one timeline. Diffs are rich blocks inline. TOC floats for navigation when content grows.
F3: Focus Shift
Temporary Split for Review

Default is full-width conversation. When an artifact needs focused attention (reviewing a diff), the layout SPLITS: artifact 60% left, conversation 40% right. When done (approve/dismiss), layout returns to full-width. The split is contextual, not permanent — triggered by clicking "review" on artifact reference cards.

KEY: No permanent panels. Split is temporary and contextual. Best for Specify phase where diffs need proper screen real estate.
