📄 Ticket Description

What: Add an authentication system with signup, login, session management, and role-based access control (RBAC).

Purpose: Enable multi-user access with proper permissions. Users should only see and modify their own resources unless granted admin privileges.

How: New auth/ module following the layered pattern (models → service → storage → middleware). Token-based auth with bns_ prefix, stored in SQLite via aiosqlite.

Success criteria: Users can sign up, log in, maintain sessions, and have role-based permissions enforced on all API endpoints.

- Session: id, user_id, token, expires_at, created_at

- Prefix: bns_
