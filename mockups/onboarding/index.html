Bonsai · Onboarding

Three-flow welcome experience: start a new project, or open an existing one and have Bonsai investigate it. Both paths converge on the same GOAL&REQUIREMENTS.md document — for new projects we draft from your idea, for existing projects we draft from the code.

Flow A · Welcome Flow B · New project (6 screens) Flow C · Existing project (investigation) Convergence: GOAL&REQUIREMENTS.md

Flow A · Welcome

Single entry point. Three doors: new / existing / pick recent.
A1 · welcome
Welcome screen
Hero with two primary CTAs (new / existing) plus a list of recent projects.

Flow B · New project

Form → guided session → G&R doc → optional architecture (chat → verify) → board with tickets.
B1 · form
New project form
Project name + goals textarea + optional doc attachment. Same fields as NewProjectScreen.tsx.
B2 · session
Guided session · live doc
Doc panel takes the wider column (chat slim on left). Real-document typography. Each new answer adds a section — most recent has a green-dot heading. No boxed cards or status tags.
B3 · doc
G&R doc · plant features
Inline-editable doc (hover any section). Big CTA card "Plant 4 v1 features → board" with checklist + secondary "Design architecture first".
B3.5 · arch chat
Architecture (optional)
Live-doc pattern for DESIGN_DOC.md. Stack + modules + scaffold preview. Skip banner always visible at top.
B3.6 · arch verify
Verify before saving
Full DESIGN_DOC.md + box-arrow architecture diagram in body. Sticky right rail: scaffold tree of 4 new files (DESIGN_DOC + 3 module READMEs), sanity checks, primary Save & continue → board.
B4 · board
Board (landing)
6-column kanban (Idea → Done). 4 freshly-planted features in Described pulsing green. FAB + first-visit tip.

Flow C · Existing project NEW

The new flow. Bonsai investigates the codebase live — we want it to be obvious what it's reading and what it will produce. End state mirrors Flow B but better: both G&R and DESIGN_DOC are drafted at once, since investigation already had everything.
C1 · detect
What we'll read
Bonsai scans the repo and shows the entry points it found (README, CLAUDE.md, package files, modules). User confirms before the deeper read.
C2 · investigation
Investigation · live
The "watch it think" screen. Tree of files with read status, current file open, observations forming on the right, live activity log, progress bar at 67%.
C2.5 · ready
Investigation · ready
Reading complete. Center pane swaps to a summary card: stats, 4 questions queued, single CTA → Clarify. The explicit handoff between investigation and conversation.
C3 · clarify
Summary & clarifying questions
Bonsai's understanding of what this project is, plus targeted questions where the code was ambiguous. User answers in chat.
C4 · G&R tab
Verify & save · 2 docs (G&R)
Tab switcher between two drafted docs. G&R tab active. Sticky rail: 6 files to create (G&R + DESIGN_DOC + 4 module stubs), sanity checks, primary Save 6 files, suggest tickets from gaps.
C4.5 · arch tab
Verify & save · 2 docs (Architecture)
Same screen, Architecture tab active. Reverse-engineered DESIGN_DOC.md: stack from pyproject.toml, 4 modules from app/, box-arrow diagram, decision log inferred from code.

Notes

Design language pulls from ../bonsai-v1-workspace.html (Darcula tokens, app frame, stepper, session chat). The new pieces are C2 Investigation live, C2.5 Investigation ready (the handoff), and C3 Clarify — they give the existing-project flow its own personality vs. the form-first new-project path.
