Implement Spec validator.py: validate spec structure, fields, and link integrity

validator.py ensures specs conform to required structure. It validates individual
spec fields (required fields present, correct types) and cross-spec link integrity
(all link targets exist, no orphaned references). Used by the service layer before
writes and during health checks.

Spec reference: backend/app/spec/README.md

Public interface to implement:

- validate_spec(spec: Spec, entry: RegistryEntry) → list[str]
    Validate a single spec's structure and fields.
    Returns list of error messages (empty = valid).
    Checks: required fields present, type is recognized, path is valid.

- validate_links(links: list[Link], entries: list[RegistryEntry]) → list[str]
    Validate link integrity across the spec graph.
    Returns list of error messages (empty = valid).
    Checks: all from_id/to_id reference existing entries, no self-links,
    link types are recognized.
