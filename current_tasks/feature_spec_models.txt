Implement Spec models.py: Pydantic data models for the spec domain

The Spec module is the core domain layer of Bonsai. models.py defines all data
structures used across the module — every other spec file imports from here.
This is the foundational dependency with no internal module dependencies.

Spec reference: backend/app/spec/README.md

Models to implement:

- Spec
    type: str                          — spec type (e.g. "module-design", "task-spec")
    content: str                       — raw file content
    metadata: dict | None              — parsed JSON object for JSON specs; None for Markdown

- RegistryEntry
    id: str                            — unique identifier
    type: str                          — spec type
    path: str                          — relative path from project root
    title: str                         — human-readable title
    status: str                        — e.g. "active", "draft", "archived"
    covers: list[str]                  — file/directory patterns this spec covers
    tags: list[str]                    — classification tags
    created: str                       — ISO date string
    updated: str                       — ISO date string

- Link
    from_id: str                       — source spec ID (serializes to "from" in JSON via Pydantic alias)
    to_id: str                         — target spec ID (serializes to "to" in JSON via Pydantic alias)
    type: str                          — relationship type (e.g. "depends-on", "parent", "implements")
    Note: Use Pydantic Field(alias="from") / Field(alias="to") since "from" is a reserved keyword

- SpecSummary
    id: str, type: str, path: str, status: str, title: str, tags: list[str]
    Lightweight listing model for list_specs()

- SpecDetail
    id: str, type: str, path: str, status: str, title: str, tags: list[str],
    content: str, links: list[Link]
    Full spec with content for get_spec()

- SpecGraph
    nodes: list[RegistryEntry]
    edges: list[Link]
    Complete hierarchy for get_graph()
