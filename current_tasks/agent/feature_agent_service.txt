Implement Agent service.py: orchestration facade for agent task management

service.py is the public facade for the Agent module. It is called by
rpc/methods/agents.py and provides the complete interface for starting tasks,
interrupting them, querying status, and relaying frontend responses to pending
futures. It coordinates runner.py and tracker.py without exposing their internals.

Spec reference: backend/app/agent/README.md (lines 46-53)

Public interface to implement:

- run_task(spec_ids: list[str], config: AgentConfig, notify: Callable) → AgentTask
    Start an agent task:
    1. Create task via tracker.create_task
    2. Load spec content via spec/service for the given spec_ids
    3. Set task status to running
    4. Launch runner.run as a background asyncio.Task
    5. On completion: set status to done, store result
    6. On error: set status to error
    7. Return the AgentTask immediately (run is async)

- interrupt_task(task_id: str) → None
    Interrupt a running task:
    1. Cancel all pending futures via tracker.cancel_futures
    2. Cancel the background asyncio.Task
    3. Set task status to error

- get_task(task_id: str) → AgentTask
    Delegate to tracker.get_task.

- list_tasks() → list[AgentTask]
    Delegate to tracker.list_tasks.

- respond(task_id: str, request_id: str, response: dict) → None
    Relay a frontend response to a pending future:
    1. Delegate to tracker.resolve_future(task_id, request_id, response)
