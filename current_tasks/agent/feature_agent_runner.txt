Implement Agent runner.py: Claude Agent SDK integration and event stream mapping

runner.py is the sole integration point with the Claude Agent SDK. It iterates
the SDK's event stream, maps SDK events to AgentEvent notifications, and handles
the canUseTool callback for both user questions and tool approvals by suspending
execution via asyncio.Future (managed by tracker.py).

Spec reference: backend/app/agent/README.md (lines 40, 84-128)

Responsibilities:
- Configure and invoke the Claude Agent SDK with spec context as system prompt
- Iterate the SDK event stream, mapping each event to an AgentEvent
- Send events to the frontend via the notify callback
- Handle canUseTool callback: distinguish AskUserQuestion vs tool approvals,
  register a Future via tracker, send a JSON-RPC request to the frontend,
  await the Future, and return the response to the SDK

Public interface to implement:

- run(task: AgentTask, spec_context: str, notify: Callable, tracker: Tracker) → AgentResult
    Execute an agent run. Async generator or coroutine that:
    1. Configures SDK with task.config settings
    2. Sets spec_context as system prompt / initial context
    3. Iterates SDK events, mapping to AgentEvent and sending via notify
    4. Registers canUseTool callback that:
       - If tool_name == "AskUserQuestion": send agent/askUserQuestion request
       - Otherwise: send agent/confirmAction request
       - Both: register Future in tracker, await it, map response to SDK return
    5. Returns AgentResult on completion

SDK Event → AgentEvent mapping (event_type):
- SDKSystemMessage (init subtype) → session_start
- SDKAssistantMessage text block / SDKPartialAssistantMessage text_delta → text_delta
- SDKAssistantMessage tool_use block → tool_call_start
- SDKUserMessage tool_result block → tool_call_end
- SubagentStart hook → subagent_start
- SubagentStop hook → subagent_end
- Notification hook → notification
- SDKCompactBoundaryMessage → compact
- Internal milestones → progress
- SDKResultMessage success → done
- SDKResultMessage error → error
- SDKResultMessage permission_denials → permission_denied

canUseTool → JSON-RPC request mapping:
- tool_name="AskUserQuestion" → send agent/askUserQuestion with request_id
  Response: AskUserQuestionResponse → PermissionResultAllow(updated_input=...)
- Any other tool_name → send agent/confirmAction with request_id
  Response: ToolApprovalResponse → PermissionResultAllow() or PermissionResultDeny(...)
