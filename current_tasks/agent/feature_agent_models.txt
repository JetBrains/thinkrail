Implement Agent models.py: Pydantic models for agent tasks, events, and interactive flows

models.py defines the data structures used across the Agent module. These are
Pydantic models for task records, run configuration, streaming events, terminal
results, and the interactive request/response types for mid-run user interactions
(questions and tool approvals).

Spec reference: backend/app/agent/README.md (lines 55-92)

Models to implement:

Core Models:
- AgentTask(id, status, spec_ids, config, session_id?, created, updated)
    Task record. status is one of: pending, running, done, error.
- AgentConfig(model, max_turns, permission_mode, stream_text)
    Run configuration passed to the SDK.
- AgentEvent(task_id, session_id, event_type, payload)
    Serializable event sent as a notification to the frontend.
    event_type enum: session_start, text_delta, tool_call_start, tool_call_end,
    subagent_start, subagent_end, notification, compact, progress, done, error,
    permission_denied.
- AgentResult(task_id, session_id, result, cost_usd, turns, duration_ms, usage)
    Terminal success result.

Interactive Request/Response Models:
- Question(question: str, header: str, options: list[QuestionOption], multi_select: bool)
    A single question with selectable options. 1-4 questions per request, 2-4 options per question.
- QuestionOption(label: str, description: str)
    A selectable option within a question.
- AskUserQuestionResponse(questions: list[Question], answers: dict[str, str])
    Response to a question request. questions passes through the original questions.
    answers maps question text to selected label. Multi-select joins labels with ", ".
- ToolApprovalResponse(behavior: "allow" | "deny", message?: str, interrupt?: bool)
    Response to a tool approval request. message is the denial reason.
    interrupt=true aborts the entire task.
