Implement RPC methods/agents.py: basic read handlers (status + list)

methods/agents.py contains the jsonrpcserver handler functions for all agent/*
JSON-RPC methods. This task covers the read-only handlers: get_agent_status and
list_agents. These follow the same pattern as methods/specs.py — each handler
delegates to AgentService and returns domain models directly.

The file also defines the _handle_errors decorator for agent-specific error
mapping, which all agent handlers (including those in later tasks) will reuse.

Spec reference: backend/app/rpc/README.md (lines 205-217, 82-96)

Handlers to implement:

- get_agent_status(**params) → AgentTask
    Handler for agent/status. Params: { taskId: str }.
    Delegates to agent_service.get_task(task_id).
    Raises TaskNotFoundError → -32011.

- list_agents(**params) → list[AgentTask]
    Handler for agent/list. Params: {} (none).
    Delegates to agent_service.list_tasks().

Error code mapping (agent-specific additions to the shared table):
- TaskNotFoundError  → -32011 "Agent task not found"
- FutureNotFoundError → -32012 "No pending request"
- KeyError/TypeError → -32602 "Invalid params"
- Other exceptions   → -32603 "Internal error"
