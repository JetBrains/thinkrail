Implement RPC methods/specs.py: JSON-RPC handlers for spec/* methods

methods/specs.py contains the jsonrpcserver handler functions for all spec/*
JSON-RPC methods. Each handler delegates to spec/service and returns domain
models directly (Pydantic models serialize to JSON automatically). Error mapping
converts domain exceptions to JSON-RPC error codes per the spec.

Spec reference: backend/app/rpc/README.md (lines 185-198, 82-96)

Handlers to implement:

- list_specs(**params) → list[SpecSummary]
    Handler for spec/list. Params: {} (none).
    Delegates to spec_service.list_specs().

- get_spec(**params) → SpecDetail
    Handler for spec/get. Params: { id: str }.
    Delegates to spec_service.get_spec(id).
    Raises SpecNotFoundError → -32001.

- create_spec(**params) → SpecDetail
    Handler for spec/create. Params: { type: str, path: str, content?: str }.
    Delegates to spec_service.create_spec(type, path, content).
    Raises ValidationError → -32003.

- update_spec(**params) → SpecDetail
    Handler for spec/update. Params: { id: str, content: str }.
    Delegates to spec_service.update_spec(id, content).
    Raises SpecNotFoundError → -32001, ValidationError → -32003.

- delete_spec(**params) → None
    Handler for spec/delete. Params: { id: str }.
    Delegates to spec_service.delete_spec(id).
    Raises SpecNotFoundError → -32001.

- get_graph(**params) → SpecGraph
    Handler for spec/graph. Params: {} (none).
    Delegates to spec_service.get_graph().

Error code mapping (from spec):
- SpecNotFoundError  → -32001 "Spec not found"
- RegistryError      → -32002 "Registry error"
- ValidationError    → -32003 "Validation error"
- KeyError           → -32602 "Invalid params"
- Other exceptions   → -32603 "Internal error"
